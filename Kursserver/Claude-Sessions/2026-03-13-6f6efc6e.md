## Implementation Plan: Preset Intro Time Slots

### Overview

The feature adds a `IsPresetIntro` boolean to `AdminAvailability`, a weekly background job that auto-generates 26 weeks of preset slots, and frontend logic that restricts the meeting type selector to "Intromote" only when a coach clicks a preset slot.

A "deleted preset" is tracked by storing slot start dates in a separate table — the cron skips dates that already have a preset slot or have been explicitly deleted.

### Design Decisions and Things to Clarify Before Coding

1. **Admin name lookup by first name is fragile.** "Victoria" and "Adam" are hardcoded as first names. If there are multiple Victorias or the name changes, the cron silently skips them. A safer design is to store admin IDs in `appsettings.json` (e.g. `"PresetSlots": { "VictoriaAdminId": 3, "AdamAdminId": 7 }`). Recommend confirming which approach to use.

2. **Merge logic conflict.** The existing POST `/api/availability` merges adjacent/overlapping unbooked slots. If a preset slot exists and an admin manually adds an adjacent availability, they'll merge — losing `IsPresetIntro = true` on the merged record. This could cause the cron to re-create the preset the following Sunday.
